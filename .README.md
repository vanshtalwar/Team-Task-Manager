3-Hour Technical Task: Smart Task Manager API + Mini UI
Objective
Build a Task Management System with:
* Clean architecture
* Optimized APIs
* Some real-world logic (not basic CRUD)

 Requirements
 1. Authentication (Basic but clean)
* JWT-based login/signup
* Fields: name, email, password
* Password hashing required

 2. Task Module (Core Logic Focus)
Task Schema:
* title
* description
* status → Todo | In Progress | Done
* priority → Low | Medium | High
* dueDate
* createdBy (userId)

 3. Smart Features (Important)
✅ A. Auto Status Logic
* If dueDate < today AND status ≠ Done
→ mark as “Overdue” (virtual or computed field)

✅ B. Prevent Invalid Updates
* Cannot move task directly:
    * Todo → Done 
    * Must go: Todo → In Progress → Done

✅ C. Priority-based Sorting API
* Custom sorting:
    * High > Medium > Low
* Not default Mongo sort → use custom logic

4. APIs to Build
Auth:
* POST /auth/register
* POST /auth/login
Tasks:
* POST /tasks → Create task
* GET /tasks →
    * Filters:
        * status
        * priority
    * Search by title
    * Pagination (limit + page)
* PATCH /tasks/:id → update task
* DELETE /tasks/:id

5. Analytics API (Important)
Create endpoint:
* GET /tasks/stats
Return:
* Total tasks
* Completed tasks
* Overdue tasks
* Tasks grouped by priority

6. Optimization (Must Have)
* Use:
    * MongoDB indexing (on status, priority)
* Avoid:
    * unnecessary queries
* Clean error handling middleware

 7. Frontend (Light – Not heavy UI)
Create a simple React UI:
* Login page
* Task list page
* Add task form
Show:
* Task list with:
    * Status
    * Priority
    * Overdue highlight (red)

 Time Expectation (3 Hours Split)
* Backend: ~2 hrs
* Frontend: ~1 hr (basic UI)

Evaluation Focus
Instead of full features, check:
* ✅ Code structure (routes, controllers, models)
* ✅ Clean validation & error handling
* ✅ Logic implementation (status flow, overdue)
* ✅ Query optimization
* ✅ Proper use of async/await
